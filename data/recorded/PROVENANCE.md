# Provenance — every recorded file in this directory

All files are byte-for-byte copies from [`github.com/vsiwach/baseten-mvp`](https://github.com/vsiwach/baseten-mvp)
(the hands-on evidence repo behind this MVP), copied 2026-07-06. Nothing here
was edited. Any number on the site carrying a `MEASURED` chip traces to one of
these files; anything that doesn't is generated by the seeded simulator and
carries a `SIMULATED` chip.

| File | Origin (in baseten-mvp) | What it is |
| --- | --- | --- |
| `chaos_drills.csv` | `benchmarks/raw/chaos_drills.csv` | 65 chaos-drill runs against live pools, 2026-07-02 → 07-04. Agent-on rows cluster at **MTTR 8.1–9.2s** (incl. the 8.8 and 9.2s runs quoted on the site); every agent-off row is `resolved=False` — the manual baseline never recovered within the drill window. |
| `episodes-live.jsonl` | `learning/episodes/live.jsonl` | 72 recorded incident episodes (policy params, trajectory, probes, outcome). The console's replay mode plays `ep-inc-0001-1783126701` (2026-07-04, MTTR 8.8s) verbatim. |
| `model_api_sweep_20260703-*.csv` (×2) | `benchmarks/raw/` | Per-request sweep of Baseten Model APIs (11 models): status, TTFT, tok/s, billed cost, tokens. Source of the `baseten-model-api` pool's measured TTFT and $/Mtok (glm-4.7: p50 TTFT 299ms, $2.78/Mtok). |
| `rate_limit_glm47_20260702-183916.csv` | `benchmarks/raw/` | The friction #10 evidence: 40 sequential requests, 28× 429 with no Retry-After. Basis of the rate-limit-coupling hazard on the Model API pool card. |
| `migration_drills.csv` | `benchmarks/raw/migration_drills.csv` | Graceful vs immediate migration drills: TTFT before/during/after, re-prefills, drain seconds. Grounds the zero-drop drain story on the release panel. |
| `live_mcp_metrics_summary_20260704-191352.json` + `_191613.json` | `benchmarks/raw/` | The friction #18 pair, read through Baseten's own MCP server: the first call (~35s after traffic) returns counter 0.0 and all-null histograms; the same call ~2 min later returns the true values (counter 7.0, full quantiles). Why this console renders "no data yet (lag)" instead of zeros. |
| `live_mcp_activation_20260704-191318.csv` | `benchmarks/raw/` | The agentic-onboarding proof: deployment activated and observed end-to-end through the Baseten MCP surface (evals/mcp-deploy in baseten-mvp). |

Numbers quoted from the friction log (warm TTFT ~330ms, TPOT ~34ms/tok on
`T4x8x32`; cold start 360.4s → 148.2s after the BDN `weights:` block) come from
deployment logs committed in baseten-mvp (`docs/FRICTION_LOG.md` #15/#17,
deployments `w52yvzr`/`qvm1v4e`). The `T4x8x32` price ($0.01504/min =
$0.9024/hr) is Baseten's published instance price
(docs.baseten.co/deployment/resources, read 2026-07-06).
