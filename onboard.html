Onboard in one prompt

You don't install this product. You paste one prompt into the coding agent you already use — Claude Code, Cursor, any MCP client — and it attaches your existing endpoints read-only, baselines every route into your ledger, and proposes the first candidate. Nothing moves traffic until you've seen the evidence.

THE ONBOARDING PROMPT — copy, paste, done

# Paste into your coding agent (Claude Code / Cursor / any MCP client).

Connect the Baseten MCP server (https://api.baseten.co/mcp) with my workspace
API key, and install the Baseten agent skills (basetenlabs/baseten-skills).

Then onboard this workspace into a reliability ledger:

1. ATTACH (read-only): register these OpenAI-compatible endpoints with a
   read-scoped key each — do not request write access:
     - https://model-XXXXX.api.baseten.co/environments/production/sync/v1   (baseten-dedicated)
     - https://inference.baseten.co/v1                                      (baseten-model-api)
     - https://myorg--qwen3-serve.modal.run/v1                              (modal-dedicated)

2. BASELINE: for each route, run a 60-request streaming bench (mix of short
   and long prompts) and record per request: TTFT, tokens/sec, status, and
   billed cost where the API reports it. Compute p50/p99 TTFT, p99 TPOT,
   goodput at my SLO (p99 TTFT ≤ 500ms, TPOT ≤ 50ms/tok), and measured
   $/Mtok = instance $/hr ÷ measured tok/s (list price for dedicated,
   billed cost for per-token APIs). Write one CSV per pool into
   data/recorded/ — these are my ledger, exportable, mine.

3. PROPOSE: rank every route × eligible-pool pair by measured $/Mtok at
   equal-or-better SLO. If an external route would hold its SLO cheaper on
   a Baseten pool, propose it as a SHADOW candidate (mirrored traffic,
   responses discarded) — with the side-by-side numbers.

4. STOP. Do not shift any traffic. Show me the ledger and the proposal;
   promotion happens only through the certify gate, with rollback held.

WHAT HAPPENS NEXT

1

Attach — read-only

Any OpenAI-compatible URL plus a read-scoped key. External clouds get a MONITOR-ONLY badge and zero control affordances — structurally, not cosmetically: the agent's decision core refuses actions on them (unit-tested).

2

Baseline → route ledger

Real SLO percentiles and measured $/Mtok per route, from streaming benches you can rerun (bench/). The ledger is yours and exports as CSV.

3

A/B + shadow

When the ledger finds a route that would hold its SLO cheaper on Baseten, a win-back card proposes a shadow: mirrored traffic, responses discarded, client untouched.

4

Certify & migrate

Parity + SLO gates on the mirrored cohort, side by side. Traffic moves only on a passing certificate, and rollback stays armed for the route's life. A refusal is the system working.
